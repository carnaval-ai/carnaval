The 7-Stage Anonymization Pipeline

Carnaval divides its operation into independent, auditability-focused modules ensuring robust detection, arbitration, and secure data substitution.

01
Ingest

S1. Intake

Loads input documents safely. Resolves file encodings with auto-fallback from UTF-8 to Latin-1. Enforces file limits up to 10 MB to prevent resource exhaustion.

02
Normalise

S2. Preprocess

Standardizes text structure. Sanitizes pipeline delimiters (pipes), collapses redundant white spaces, strips BOMs, and detects the document language (French, English, German, or Spanish).

03
NER & Regex

S3. Detect

Dispatches active recognizers: zero-shot neural entity recognition via GLiNER for open classes (people, organizations), language-specific regexes (VAT, IBAN, phones), and strict denylists.

04
Arbitrate

S4. Resolve

De-duplicates and arbitrates overlapping entities. Employs scoring policies where longer spans and higher recognizer confidence take precedence to resolve conflicts.

05
Encrypt

S5. Mask

Substitutes raw text values with semantic placeholders (e.g. [FOURNISSEUR_1], [CLIENT_NAME]). Writes sensitive maps into a local vault encrypted via AES-256-GCM.

06
Export

S6. Output

Serializes the masked document and encrypted vault. Supports 6 target formats: plain text, JSON, JSONL, XML, CoNLL, and visualization-ready HTML.

07
Reversible

S7. Reinject (Post-LLM Processing)

Crucial reversing stage. Parses structured XML or JSON payloads returned by cloud LLMs, decrypts the local vault, and matches semantic placeholders to restore original values into the final business output without cloud leakage.

True Zero-Cloud Moat

By encrypting masking tables locally with AES-256-GCM, plain text PII never leaves your computing boundary. The cloud LLM only processes standardized, non-identifying semantic tokens.

Zero-Shot NER (GLiNER)

Leverages GLiNER models for customizable entity detection. No heavy spaCy models or Presidio installations needed. Supports on-the-fly definition of custom target tags through simple configuration.

Production API Contract

Standardized interfaces designed for easy workflow integration. Features direct pipeline execution and a command-line interface suitable for automation inside Dockerized orchestrations.

Developer API & CLI Quick Start

Get up and running with Carnaval in a matter of seconds. Standard Python usage follows familiar patterns designed for modern workflows.

example_integration.py
from carnaval.pipeline import run_anonymization
from carnaval.stages.s7_reinject import reinject_original_values

# 1. Mask PII local-first using a business profile
masked_doc, vault, metadata = run_anonymization(
    input_path="inbox/sample_invoice.txt",
    outbox_dir="outbox/",
    vault_password="secure_aes_encryption_key_32_chars",
    profile="invoice",
    use_gliner=False
)
print(masked_doc.anonymized_text)
# Output: "Billing to [CLIENT_NAME] from [FOURNISSEUR_1]... Order: [REF_COMMANDE_CLIENT_1]"

# 2. Safely process the masked document in the cloud via any LLM
llm_response_json = {
    "provider": "[FOURNISSEUR_1]",
    "client": "[CLIENT_NAME]",
    "order_reference": "[REF_COMMANDE_CLIENT_1]",
    "total_ht": 3000.0
}

# 3. Restore original PII values back into the structured output
restored_json = reinject_original_values(
    structured_data=llm_response_json,
    vault=vault
)
print(restored_json)
# Output: {"provider": "Initech", "client": "Acme Corp", "order_reference": "FAC-2024-0892", "total_ht": 3000.0}
